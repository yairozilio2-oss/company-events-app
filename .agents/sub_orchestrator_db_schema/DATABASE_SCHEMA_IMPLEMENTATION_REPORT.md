# Database Schema Implementation Report

**Feature:** Registration

## Schema Design Overview
- **users** table stores core account data:
  - `id` (primary key, auto‑increment)
  - `email` (unique, not null)
  - `password_hash` (not null)
  - `verification_token` (unique, nullable)
  - `is_verified` (boolean, default false)
  - `created_at`, `updated_at` timestamps
- **profiles** table stores optional user profile fields and has a one‑to‑one relationship with `users`:
  - `user_id` (primary key, foreign key to `users.id` with cascade on delete)
  - `first_name`, `last_name`, `phone` (all nullable)
